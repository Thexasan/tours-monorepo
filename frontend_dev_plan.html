Базовая архитектура
Структура папок App Router важно
app/(public)/ — публичные страницы. app/(auth)/ — кабинеты за логином. app/admin/ — панель админа. Каждая группа имеет свой layout.tsx.
app/(public)/layout.tsx · app/(auth)/layout.tsx · app/admin/layout.tsx
API-клиент (Axios + TanStack Query)
Базовый axios instance с baseURL из env, interceptor для JWT токена, обёртка для обработки ошибок. QueryClient провайдер в корневом layout.
↗ AI: попроси написать базовый apiClient.ts с refresh token логикой
Auth стор (Zustand)
useAuthStore: user, token, роль, isLoading. Персистентность через zustand/middleware persist (localStorage). Хук useRequireAuth для защиты страниц.
i18n фундамент (next-i18next)
Настройка next-i18next.config.js, структура переводов locales/ru, locales/en, locales/tj. Middleware для определения языка из URL (/ru/, /en/). Пока заполняешь только русский.
locales/ru/common.json · tours.json · auth.json · dashboard.json
Базовые компоненты Layout
Navbar (логотип, навигация, языковой switcher, кнопки входа), Footer (ссылки, контакты, соцсети), PageWrapper с max-width контейнером.
После фазы 1: проект запускается локально, Auth API подключён, базовая навигация работает. Это фундамент — не торопись, здесь закладываются паттерны на весь проект.

Недели 3–6~80 ч
Главная страница
Hero + поисковый блок
Форма поиска: куда (select с городами), когда (DateRangePicker из shadcn), количество человек. При сабмите — редирект на /tours с query-параметрами.

Блок горящих туров
Горизонтальный скролл на мобиле, grid на десктопе. TourCard компонент: фото (next/image), название, цена в выбранной валюте, кнопка. Данные через SSR (fetch на сервере).
Блок "Как поехать бесплатно"
3 шага с иконками (shadcn Icon или Lucide). Статичный блок, никакого API.
Блок последних отзывов
Карточки с аватаром (из профиля), именем, текстом и фото из поездки. SSR данные. Ссылка на профиль пользователя.
Каталог туров
Страница каталога с фильтрами SEO важен
URL-based фильтры (?stars=4&food=all-inclusive&price_max=500) для SEO. Sidebar с фильтрами на десктопе, drawer на мобиле. Сортировка: дешевле, популярнее, новее.
app/(public)/tours/page.tsx — серверный компонент с searchParams

TourCard компонент
Переиспользуемый: фото (Cloudinary + next/image), название, звёзды отеля, питание, цена с конвертацией валюты, кнопка "Подробнее". Skeleton лоадер для loading state.
Пагинация
URL-based (?page=2), shadcn Pagination компонент. Infinite scroll — только если явно нужен заказчику, иначе классическая пагинация проще.
Страница тура
Фотогалерея средняя сложность
Главное фото + thumbnails. Lightbox при клике (react-photo-view или embla-carousel). Все изображения через next/image с Cloudinary URL-трансформациями (автоматический webp, ресайз).
Детальное описание тура
Программа (accordion по дням), что включено / не включено (списки с иконками ✓ и ✗), характеристики (звёзды, питание, длительность). Всё из JSONB поля нужного языка.
Форма заявки важно
Sticky кнопка на мобиле. Modal или sidebar с формой: имя, телефон, email, комментарий. React Hook Form + Zod. После submit — toast "Менеджер свяжется с вами".

Блок отзывов на странице тура
Только апрувнутые отзывы этого тура. Карточка: аватар, имя (ссылка на профиль), рейтинг (звёзды), текст, фото (grid миниатюр). Если отзывов нет — пустой стейт.
SEO мета-теги
generateMetadata() для каждого тура: title, description, Open Graph (og:image из первого фото тура). Structured data (JSON-LD) для туристического продукта.
Auth страницы
Логин / Регистрация / Сброс пароля
Три страницы с формами React Hook Form + Zod. После логина — редирект по роли: клиент → /dashboard, партнёр → /partner, админ → /admin.
Реферальная ссылка (?ref=xxx) должна сохраняться при переходе на форму заявки. Не теряй ref параметр при навигации — прокидывай его через весь flow до момента submit.

Мои туры
Список купленных туров. Каждый элемент: фото тура, название, даты, статус бронирования (бейдж: В работе / Оплачен / Завершён). Пустой стейт с CTA "Найти тур".
Реферальный блок ключевая фича
Уникальная ссылка + кнопка копирования (clipboard API + toast). Кнопки шеринга: Telegram, WhatsApp (window.open с готовыми URL). Счётчик прогресса.
Progress bar "до бесплатного тура" UX важен
Анимированный прогресс: referred_count / referral_threshold × 100%. Текст "Вы привели N человек. Осталось X". При 100% — поздравительный баннер. Данные из API.

Доступна только для завершённых туров. Multi-upload: drag-and-drop зона, превью фото, удаление. Textarea для текста. После submit — статус "На модерации" вместо формы.

Фаза 4 — Кабинет партнёра
Недели 10–12~60 ч
Layout кабинета партнёра
Отдельный layout от клиентского. Sidebar: Дашборд, Финансы, Мои ссылки, Настройки. Отдельный Protected route с проверкой роли partner.
Дашборд с графиками средняя сложность
Recharts: LineChart переходов по дням, BarChart продаж по месяцам. Метрики вверху: всего переходов, регистраций, покупок, заработано. Фильтр периода (7д / 30д / 90д / всё время).

Страница финансов
Баланс (большой текст сверху), таблица транзакций (дата, тур, сумма покупки, 5% комиссия, статус). Сортировка по дате. Экспорт в CSV (простой: blob download).
Форма вывода средств
Поля реквизитов (карта/счёт), сумма к выводу (не больше баланса — валидация Zod), кнопка "Запросить вывод". После отправки — статус "Запрос на рассмотрении".

Список ссылок партнёра (можно создавать под разные туры / кампании). Статистика по каждой ссылке: переходы, регистрации, продажи.
Фаза 5 — Админ-панель
Недели 13–15~70 ч
Используй react-admin или напиши кастомно на shadcn/ui DataTable. React-admin сэкономит ~30ч на типовых таблицах, но потребует время на изучение его паттернов. Если хорошо знаешь React — кастомно на shadcn/ui быстрее и гибче.
Управление турами
Таблица туров с поиском
shadcn DataTable с колонками: фото, название, цена, статус (активен/нет), referral_threshold, действия. Поиск по названию, фильтр по статусу.
Форма создания/редактирования тура сложная
Табы RU / EN / TJ для каждого текстового поля. Multi-upload фото (порядок drag-and-drop). Поля: название, описание, программа, включено/не включено, цена (в TJS), звёзды, питание, referral_threshold.

Обработка заявок
Таблица заявок триггерная точка
Фильтры по статусу, туру, дате. Колонки: клиент, тур, дата, сумма, статус, реферер. При смене статуса на "Оплачена" — модальное подтверждение ("Это запустит начисление бонусов").
Смена статуса заявки
Select со статусами только в правильном направлении (нельзя вернуть назад). Confirm modal перед "Оплачена". После смены — optimistic update таблицы.
Модерация UGC
Очередь модерации
Бейдж-счётчик непроверенного контента в navbar. Карточки с фото и текстом отзыва. Кнопки: Одобрить / Отклонить (с причиной). Фильтр: все / ожидают / одобрено / отклонено.
Пользователи и финансы
Управление партнёрами
Список заявок на партнёрство (статус pending), кнопка апрув/отклонить. Список активных партнёров с их балансом и статистикой.
Выплаты партнёрам
Таблица запросов на вывод: партнёр, сумма, реквизиты, дата. Кнопка "Подтвердить выплату" → статус меняется, на баланс партнёра начисляется минус.
Курсы валют
Простая страница: три поля TJS→USD, TJS→RUB с текущими значениями, кнопка сохранить. Обновляешь вручную когда нужно.
